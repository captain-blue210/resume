#### 新規サービス開発

- **プロジェクト規模**
  - メンバー数：13 名
- **担当業務**
  - 開発、テスト、リリース、業務改善
- **業務概要**

  - Nuxt.js でのフロントエンド開発
  - 既存ライブラリのアップデート
  - 各環境の構築
  - 自動テスト整備の提案と主導
  - モブプログラミング導入の提案と主導
- **業務詳細**

全国規模の新規開発プロジェクトに参画。
バックエンドからフロントエンド、インフラまわりまで一気通貫でタスクを担当し、Nuxt.js 未経験メンバーのヘルプも担当しました。また、教育が得意なことを活かし、初年度から新卒研修も一部担当しています。

プロジェクトでは、テストが手動かつ観点もメンバーによって様々だったこともあり、リグレッションによるバグが多く下流工程で発生していました。この問題に対処するため、テスト改善プロジェクトを提案・主導。

まず、全体の目標およびスケジュールを作成し、マネージャーやメンバーと合意をとりました。
次にこれまで発生したバグのチケットを分析し、バグが多く発生している箇所と、どのようなテストをしていればよかったのかを明らかにしました。
こうして突き止めた場所からテストを整備しました。

また、隔週でテストに特化した MTG を開いています。そこでは直近で起きたバグを取り上げメンバーで「どうすればそのバグを防げたか？」を議論したり、テストを書く際にでてきた不明点や疑問点を解消しメンバーのテストスキルを高める場としています。

テスト改善プロジェクトについては、現在も進行中でまだまだテストのカバー率は高くありません。現時点の成果としては以下になります。

- テストを整備することで得られる具体的なメリットについてチーム内で認識をあわせることができた
- レビュー依頼事にテストケースとその結果を記載するルールができた
  - 現時点では手動テストと結果のキャプチャが主だが、少しずつ自動テストで書けるようにしていきたい
- バグについて自分の担当以外の内容も知ることができ、メンバーのテスト精度が向上した

### 株式会社コマースニジュウイチ（2016/04〜2021/01）

こちらの会社での経験は基本的に各プロジェクトで似通っているため、全体を通して意識しておこなった貢献などをまとめて記載します。

- 暗黙知のドキュメント化

各プロジェクトを通して意識していたのは、知見や手順をドキュメント化して属人性を最小限にすることです。
携わったプロジェクトは、すでに数年以上は保守・運用されたものが多くドキュメント化されていない知識も存在しました。

知識が属人化していると、その人がいないと何もできないということもあるため、ドキュメントになっていない知識があれば積極的に記載してメンバーに周知していました。また、既存の内容が古くなっている場合にはアップデートもしていました。

- 新人研修プロジェクト

製品の最新バージョンに合わせた新人研修用の資料作成に携わりました。
研修での相手に合わせた説明や教え方を評価され、 製品のライセンス購入したお客様への説明を任せていただける機会をいただきました。

また、コロナの中で新卒研修も担当しました。

- 両者ともに慣れないリモートでの実施であった
- 新卒の方が 1 人のみでなかなか緊張がとれなかった

こうした環境で、なかなかうまく質問をしてもらえないという問題が発生しました。
そこで、以下を実施したところ、質問の回数を増やすことができ、さらには質問のレベルも他の研修担当者が驚くくらいに上げることができました。

- リモートワークでの研修でコミュニケーションを取るために以下を実施
  - 毎日 30 分、進捗確認と雑談 MTG を設定
  - 質問しやすいようにするだけでなく、こちらから積極的に分からないところがないかを確認する
- 緊張して質問がうまくできない
  - 自分の失敗話や新卒のときの様子を話すなどして緊張を和らげる
  - 質問の仕方のテンプレートを作成し、使い方を指導
  - なにが分からないのか分からない状態になってしまったら、どこまでを理解したかを 1 つずつ順を追って確認する
    - 分からなくなってしまった箇所を把握して指導をおこなう

## 興味関心

これまで EC に特化したキャリアだったため、これからは EC 以外の分野に挑戦したいと考えています。
現在は、主に教育やヘルスケア分野に興味があります。

なぜ教育に興味があるのかというと、2 つの理由があります。

- 勉強が好きで、学習方法をアップデートしていくのに強い関心がある
- 教育格差への問題意識

私自身が勉強することを非常に楽しく感じる人間です。さらにいえば、どうすればより効率よく勉強できるのかを考えアップデートしていくのが趣味といっても過言ではありません。情報をアップデートしていると、こうした知見を取り入れた教育コンテンツを作成できれば、自分だけでなくもっと多くの人の役に立てるのではと考えるようになりました。

加えて、現在の社会では親の資本格差がそのまま子供の教育格差に反映されているという現状があります。このような問題の解決に技術を用いて挑戦してみたいと考えています。

ヘルスケア分野にも興味がありますが、こちらは身の回りに医療関係者が多かったり、妻が持病の関係で通院する機会が多いためです。
医療機関で働く方たちや病院に通う人の話を聞くと、その大変さに驚きます。IT 技術を使って大変さを減らしたりよりよくすることに貢献したいと思っています。
